## Purpose

**Pipeline purpose**
> A flexible automated chart-race video pipeline that adapts data sources, chart styles, and per-video features to the data; ingests performance signals from YouTube/X to refine future ideas and the pipeline itself; with the end goal of generating passive revenue from viewership.

**World-stats channel purpose**
> A YouTube/X channel of short-form videos turning world data — economic, financial, geopolitical, demographic, environmental, and beyond — into era-spanning country-vs-country (or entity-vs-entity) races, leveraging historical context and "aha" shifts to retain a general curious-viewer audience.

**Sports-stats channel purpose**
> A YouTube/X channel of short-form videos turning soccer statistics — pulled from any suitable source across leagues, competitions, and stat types — into player-vs-player (or team-vs-team) races, leveraging era nostalgia, milestone moments, and ranking surprises to retain a soccer-fan audience.

## Requirements baseline

**Shared render engine**
1. Render value-weighted bar-chart races at 30fps with smoothstep-blended gliding rank transitions.
2. Support pluggable data sources via `DataSource` ABC and asset providers via `AssetProvider` ABC.
3. Support multiple themes (currently `glass_dark`, `glass_dark_drift`, `glass_dark_lava`, `glass_dark_black`, `flat_light`).
4. Support a right-side "spotlight" callout for non-top-N entities with notable rate-of-change.
5. Support per-row sparklines, rank-change flash (green/red), rounded flag corners, header trend line.
6. Support per-row "rate" badges and "retired" flagging for monotonically non-decreasing series (opt-in).
7. Support typography control (global `font_scale` + per-section size/weight overrides).
8. Support background animation (radial drift, lava-blob).
9. Support intro animations (back-ease-out bounce on title, rows, trend line).
10. Auto-size name-box width per video based on the longest top-N name that will ever appear.

**Pipeline orchestration**
11. Per-capita transform (divide indicator by `SP.POP.TOTL`).
12. Accumulated/cumulative transform (`cumsum` after per-capita).
13. Single-frame and multi-frame PNG previews for fast layout iteration.
14. Layout validation flag (`--validate-layout`).
15. Refetch flag to redownload source data, flags, population.

**World-stats data layer**
16. Fetch World Bank indicators via `wbgapi`, cache to CSV.
17. Fetch flags from flagcdn.com with 5:3 aspect normalization.

**Sports data layer**
18. Scrape fbref for cross-league career goals and full PL history (Firecrawl + rate-limit + HTML cache).
19. Resolve fbref player IDs with disambiguation prompts on collisions.
20. Fetch and cache headshot images via the ssref proxy; run rembg/U²-Net to remove backgrounds.
21. Provide a separate sports pipeline that skips per-capita / accumulated / narration / WB cache logic.

**Narration**
22. Generate LLM scripts (Anthropic) with stat-pack + frame-keyed event timeline.
23. Generate variant pools (5 hooks / 3 middles / 3 endings) with no-spoiler discipline.
24. Synthesize TTS via ElevenLabs with per-clip SHA256 caching.
25. Mux narration audio onto the rendered mp4 (ffmpeg, `-c:v copy`).
26. Archive previous narration takes on every regeneration.
27. Mix background music with ducking under voice (ffmpeg).

**Big-mover curation**
28. Extract candidate big-mover events to `cache/big_movers.json` for human/LLM review.
29. Render only events with `keep:true` from the curated JSON when configured.

**Posting / X**
30. Hand-written X post drafts in repo (`x_post_*.md`).

**Missing capabilities (to be built)**
31. Auto-post finished videos to YouTube and X.
32. Ingest YouTube/X performance metrics to refine future video ideas and pipeline parameters.
